Jakarta Redirect Inspection Item

Vulnerability Type

Open Redirect is a common vulnerability where an application performs redirections based on user-provided URLs. Attackers can exploit this to redirect users to malicious websites, leading to phishing attacks and data leakage.

What This Item Checks

This item checks the use of Response.seeOther() and Response.temporaryRedirect() methods in Jakarta RESTful Web Services, specifically focusing on whether the URL parameter is derived from unvalidated user input.

Inspection Logic

  • If these methods use a URL parameter that is derived from unvalidated user input, it is flagged as potentially vulnerable to open redirects.
  • Warnings are displayed at insecure code locations, along with suggestions for fixing the issues.

Fix Recommendation

Developers are advised to validate user input thoroughly when handling redirect target URLs, ensuring that only legitimate redirect targets are allowed. This prevents users from being redirected to malicious websites.

Quick Fix

This item provides quick fix suggestions to help developers replace unsafe URL handling with validated inputs.

Example

For instance, if the code contains the following unsafe redirect: 


Response.seeOther(URI.create(userProvidedUrl));  // Unsafe

It is recommended to validate userProvidedUrl to ensure only trusted URLs are allowed.

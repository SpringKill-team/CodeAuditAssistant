Hardcoded Credentials Inspection Item

Vulnerability Type

Hardcoded Credentials refer to sensitive information such as passwords, keys, and tokens that are directly embedded in the source code. These hardcoded credentials can be easily extracted by attackers, leading to unauthorized access and severe security risks, especially if the codebase is exposed or leaked.

What This Item Checks

This item checks for sensitive information in the following patterns:

  • Checks for keywords like password, passwd, pwd, secret, and token in the code.
  • Checks for hardcoded passwords in database connection strings.
  • Monitors common Java classes and methods such as java.util.Hashtable.put() to ensure sensitive data is not passed.
  • Uses entropy analysis to detect possible hardcoded credentials based on the complexity of the strings.

Fix Recommendation

Developers are advised not to hardcode passwords, keys, or other sensitive information directly in the code. Store such information securely in configuration files or environment variables, and load them during application startup. Use encryption or secure management tools to manage and protect these sensitive credentials.

Quick Fix

This item provides quick fix suggestions to help developers identify and replace hardcoded credentials, and encourages storing sensitive information in a more secure manner.

Example

For instance, if the code contains the following unsafe hardcoded credentials: 


String password = "mySecretPassword123";  // Unsafe

It is recommended to change it to: 


String password = System.getenv("DB_PASSWORD");  // Safe

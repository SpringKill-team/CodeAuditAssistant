Jakarta 重定向检查项

漏洞类型

开放重定向（Open Redirect） 是一种常见的漏洞，当应用程序根据用户输入的 URL 进行重定向时，攻击者可能利用这一点将用户引导到恶意网站。此类漏洞可能导致钓鱼攻击和用户信息泄露。

检查了什么内容

该条目检查了 Jakarta RESTful Web Services 中 Response.seeOther() 和 Response.temporaryRedirect() 方法的调用，尤其是传入的 URL 参数是否来自未经验证的用户输入。

检查逻辑

  • 如果检测到这些方法的 URL 参数使用了未经验证的用户输入，则认为存在潜在的开放重定向风险。
  • 工具会在不安全的代码位置显示警告，并提供修复建议。

修复建议

建议开发者在处理重定向目标 URL 时，对用户输入进行严格的验证，确保只允许合法的重定向目标，避免用户被重定向到恶意网站。

快速修复

该条目提供了快速修复建议，帮助开发者替换不安全的 URL 处理方式，并提醒进行输入验证。

相关示例

例如，如果代码中存在以下不安全的重定向调用： 


Response.seeOther(URI.create(userProvidedUrl));  // 不安全

建议对 userProvidedUrl 进行验证，确保仅允许可信的 URL。

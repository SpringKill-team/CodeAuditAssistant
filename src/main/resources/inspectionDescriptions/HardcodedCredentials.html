硬编码凭据检查项

漏洞类型

硬编码凭据 是指在代码中直接写入敏感信息，如密码、密钥、令牌等。这些硬编码的信息可能被攻击者轻易获取，进而导致系统被未授权访问。此类问题可能会导致严重的安全风险，尤其是在代码库公开或泄露的情况下。

检查了什么内容

该条目检查了以下模式下的敏感信息：

  • 是否在代码中使用 password、passwd、pwd、secret、token 等关键词。
  • 是否在数据库连接字符串中使用硬编码的密码。
  • 检查了常见的 Java 类和方法，如 java.util.Hashtable.put()，确保没有传递敏感信息。
  • 使用熵值（entropy）检测密码或密钥的复杂度，以识别可能的硬编码凭据。

修复建议

建议开发者不要在代码中直接硬编码密码、密钥或其他敏感信息，而是将这些信息存储在安全的配置文件或环境变量中，并在应用启动时加载。使用加密或安全管理工具来管理和保护这些敏感信息。

快速修复

该条目提供了快速修复建议，帮助开发者识别并替换硬编码的凭据，并提醒将敏感信息移到安全的存储位置。

相关示例

例如，如果代码中存在以下不安全的硬编码凭据： 


String password = "mySecretPassword123";  // 不安全

建议将其修改为： 


String password = System.getenv("DB_PASSWORD");  // 安全
